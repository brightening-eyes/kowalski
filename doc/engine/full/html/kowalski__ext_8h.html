Detailed Description

This header contains extensions to the API defined in kowalski.h. These extensions are kept in a separate file because they use C style constructs (like structs and pointers to variables and functions) and are not part of the language bindings.

typedef void(* kwlDSPCallback)(float *inBuffer, int numChannels, int numFrames, void *data)

A callback that gets invoked to pass sample buffers to a DSP unit. Only called from the mixer thread. Perform manipulation of variables shared between the engine and mixer threads only in the updateDSPMixerCallback and updateDSPEngineCallback callbacks set in the kwlDSPUnit struct.

Parameters:
numChannels The number of channels of the buffer to process.
numFrames The number of frames of the buffer to process.
data The user data associated with the DSP unit.

typedef void(* kwlEventStoppedCallack)(void *userData)

Called when an event instance stops playing. The callback is invoked on the application thread. An event instance could stop playing for any of the following reasons:

  • The end of the last audio data item was reached.
  • kwlEventStop was called.
  • Data referenced by the event was unloaded.
  • The event instance was stolen during one-shot playback.
Parameters:
userData Optional user data.

kwlEventHandle kwlEventCreateWithBuffer ( kwlPCMBuffer *  buffer,
kwlEventType  eventType 
)

Creates a freeform event from a given PCM buffer. The buffer passed to this method is not released along with the event. Events created using this function exist in parallel with any loaded engine data and wave banks.

Error codes:

  • KWL_ENGINE_IS_NOT_INITIALIZED if the Kowalski engine has not been initialized.
  • KWL_POSITIONAL_EVENT_MUST_BE_MONO if the specified audio file is stereo and eventType is KWL_POSITIONAL.
  • KWL_INVALID_PARAMETER_VALUE if the number of frames is less than 1, if the number of channels is not 1 or 2 or if buffer the audio buffer is NULL.
Parameters:
buffer A pointer to the PCM buffer to create the event from.
Returns:
An event handle corresponding to the created event or KWL_INVALID_HANDLE if an error occurred.

Loads PCM (i.e uncompressed) audio data from a file. Supported file formats are au, wav and aiff. This function can be called when the engine is not initialized. Audio data loaded using this function can be released with kwlPCMBufferFree.

Error codes:

  • KWL_FILE_NOT_FOUND if the specified audio file cannot be found.
  • KWL_UNKNOWN_FILE_FORMAT if the given file is of an unknown format.
  • KWL_UNSUPPORTED_ENCODING if the audio file format is recognized but the encoding is unsupported.
Parameters:
audioFilePath The path of the audio file to create the event from.
buffer A pointer to a kwlPCMBuffer struct to populate with audio data.
